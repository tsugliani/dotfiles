# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project overview

Chezmoi-managed dotfiles repository for macOS and Debian 13 (Trixie). Uses Go templates with conditional 1Password integration. Source directory: `~/.local/share/chezmoi/`.

## Key architecture decisions

- **Single `.zshrc`, no `.zshenv`.** All shell configuration (environment, brew, oh-my-zsh, aliases, integrations) lives in `dot_zshrc.tmpl`. Brew shellenv is sourced early in `.zshrc` before oh-my-zsh (which resets PATH).
- **Linux packages: apt-first, Homebrew for extras.** Most tools installed via apt; only neovim uses Homebrew (for newer version). Package lists are in `.chezmoidata/settings.yaml` under `packages.linux.apt` and `packages.linux.homebrew`.
- **macOS packages: Homebrew only.** All packages via brew under `packages.darwin.homebrew`.
- **1Password is fully optional.** Controlled by a single `use_1password` boolean. When false, all templates fall back to local SSH keys and static config. When true, SSH agent, git signing, and secrets are managed by 1Password.
- **Oh My Zsh and tmux plugins are always user-local.** `run_onchange_after_05` clones Oh My Zsh to `~/.oh-my-zsh`, `zsh-autosuggestions`/`zsh-syntax-highlighting` into its `custom/plugins/`, and TPM + Catppuccin into `~/.tmux/plugins/`. If any of these paths is a symlink (e.g. a leftover `/usr/share` pointer), it is removed and replaced with a real clone.
- **Catppuccin Mocha** is the theme for both tmux and neovim. Tmux loads it via `run ~/.tmux/plugins/catppuccin/catppuccin.tmux`.
- **Neovim bootstrap is Lua-only.** `config/lazy.lua` auto-clones lazy.nvim on first launch. The `run_onchange_after_20` script just runs `Lazy! sync`.
- **Shared Homebrew on Linux needs explicit permission setup.** `run_onchange_after_11` sets `chgrp -R users`, `chmod -R g+rwX`, and the setgid bit on all directories under `/home/linuxbrew/.linuxbrew` so every user in the `users` group can run `brew install`. Git `safe.directory` for the Homebrew repo is set in `.gitconfig` (Linux only).

## File naming conventions (chezmoi)

- `dot_` prefix -> `.` in target (e.g., `dot_zshrc.tmpl` -> `~/.zshrc`)
- `private_dot_` prefix -> private file with `.` prefix
- `symlink_` prefix -> symlink in target
- `.tmpl` suffix -> Go template, rendered by chezmoi
- `run_once_before_` -> runs once before file management
- `run_onchange_before_` -> re-runs when rendered content changes, before file management
- `run_onchange_after_` -> re-runs when rendered content changes, after file management

## Template variables

All custom data comes from two sources:
1. `.chezmoidata/settings.yaml` — defaults and package lists
2. `~/.config/chezmoi/chezmoi.toml` — user-specific overrides (generated by `chezmoi init` from `.chezmoi.toml.tmpl`)

Key variables: `.git.name`, `.git.email`, `.use_1password`, `.ssh.personal_pubkey`, `.op.vault`, `.op.ssh_personal_item`, `.op.ssh_zpods_item`, `.op.github_personal_item`, `.packages.linux.apt`, `.packages.linux.homebrew`, `.packages.darwin.homebrew`.

Built-in: `.chezmoi.os` (`linux` or `darwin`).

## Common tasks

```bash
chezmoi apply              # Apply all changes
chezmoi apply --dry-run    # Preview changes
chezmoi diff               # Show diff of pending changes
chezmoi execute-template < file.tmpl  # Test template rendering
chezmoi state delete-bucket --bucket=scriptState  # Reset script run state
chezmoi init               # Re-run interactive prompts
```

## Testing changes

1. Create a test user: `sudo useradd -m -s /bin/zsh -G sudo,users testuser`
2. Set password: `echo 'testuser:VMware1!' | sudo chpasswd`
3. Add sudoers: `echo 'testuser ALL=(ALL) NOPASSWD: ALL' | sudo tee /etc/sudoers.d/testuser && sudo chmod 440 /etc/sudoers.d/testuser`
4. Copy source: `sudo mkdir -p /home/testuser/.local/share/chezmoi && sudo cp -r ~/.local/share/chezmoi/. /home/testuser/.local/share/chezmoi/ && sudo chown -R testuser:testuser /home/testuser/.local`
5. Test: SSH in as testuser, run `chezmoi init && chezmoi apply && source ~/.zshrc`
6. Cleanup: `sudo pkill -u testuser; sudo userdel -r testuser; sudo rm -f /etc/sudoers.d/testuser`

## Gotchas

- **Brew shellenv must be in `.zshrc`**, not just `.zshenv`, because oh-my-zsh resets PATH. Also needed for `source ~/.zshrc` to work.
- **`brew update` fails** if Homebrew repo has no git remote or `safe.directory` is not set — the install script handles this with a remote check before running `brew update`.
- **`range` on nil lists** crashes Go templates — always use `| default list` with range loops.
- **`.chezmoi.toml.tmpl` must use `dig`** (not direct map access) for nested keys, since they may not exist on first `chezmoi init`.
- **1Password CLI is a brew cask** (`brew install --cask 1password-cli`), not a formula.
- **Chezmoi `run_*` scripts execute in subprocesses** — they cannot reload the parent shell.
- **`run_once` vs `run_onchange`**: Use `run_onchange` when the script's behavior depends on template variables that may change (e.g., 1password flags). `run_once` only runs once ever, regardless of content changes.
- **`compinit` caching** uses `date -r` (GNU coreutils) not `stat -f` (BSD-only) for Linux compatibility.
- **`skip_global_compinit=1`** must be set before sourcing oh-my-zsh to prevent double compinit.
- **Symlink migration**: `run_onchange_after_05` detects and replaces symlinks (e.g. old `/usr/share/oh-my-zsh` pointers) with real clones. Adding any new `clone_if_needed` call to that script changes its hash and triggers a re-run across all users on next `chezmoi apply`.
- **Homebrew setgid is slow on large installs**: `run_onchange_after_11` runs `find … -type d -exec chmod g+s` which touches every directory under `/home/linuxbrew/.linuxbrew`. It only re-runs when its template content changes, so it's a one-time cost per user.
